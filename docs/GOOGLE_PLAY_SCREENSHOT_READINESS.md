# GOOGLE_PLAY_SCREENSHOT_READINESS

## 2026-06-17 홈 화면 디자인 리프레시 후 확인 필요

- 홈 화면이 `고요한 밤의 운세 다이어리` 콘셉트로 1차 리프레시되었습니다.
- Google Play 스토어 스크린샷은 디자인 변경 후 새 홈 화면 기준으로 다시 제작해야 합니다.
- 실제 스토어 스크린샷 이미지 파일은 아직 생성하지 않았습니다.
- 실제 Google Play Console 스크린샷 업로드는 진행하지 않았습니다.
- 스크린샷 제작 전 모바일 폭에서 CTA, 빠른 메뉴, 개인정보 안내, 광고 placeholder가 자연스럽게 보이는지 확인해야 합니다.
- 금지 문구, 민감정보 노출, 저장 기능의 이전 명칭 사용 여부를 다시 점검해야 합니다.

- 문서 경로는 `docs/STORE_SCREENSHOT_SAMPLE_PROFILE_SCREEN_QA_RESULT.md`입니다.
- 실제 캡처 이미지는 후속 단계에서 제작합니다.

## 테스트용 샘플 프로필 기준

- 스토어 스크린샷 촬영 전 테스트용 샘플 프로필 기준을 확인합니다.
- 문서 경로는 `docs/STORE_SCREENSHOT_SAMPLE_PROFILE.md`입니다.
- 실제 사용자의 생년월일, 출생시간, 성별은 스크린샷에 노출되지 않아야 합니다.
- 실제 스크린샷 이미지는 아직 생성하지 않았습니다.

## 샘플 프로필 문구 정합성

- 스토어 스크린샷 샘플 프로필 문서의 기능명 표기를 `저장한 풀이`로 통일했습니다.
- 피해야 할 문구 예시는 `투자하면 성공합니다.` 기준으로 정리했습니다.
- 실제 스크린샷 촬영 전 샘플 문구와 금지 문구가 스토어 등록 정보와 충돌하지 않는지 확인해야 합니다.

이 문서는 하루풀이 Android 앱을 Google Play에 제출하기 전 스토어 스크린샷 준비 항목을 정리한 문서입니다.
이번 PR은 스크린샷 준비 문서화가 목적이며, 실제 스크린샷 이미지 생성, Google Play Console 입력, production 앱 로직 변경은 진행하지 않습니다.

## 1. 목적

- Google Play 등록에 필요한 스크린샷 후보 화면을 정리합니다.
- 앱의 핵심 기능을 스토어에서 이해하기 쉽게 보여줄 화면을 선정합니다.
- 운세/사주 앱 특성상 과장, 단정, 불안 조장 문구를 피하기 위한 기준을 정리합니다.
- 실제 스크린샷 촬영은 Android 실제 QA 또는 에뮬레이터 환경 준비 이후 진행합니다.

## 2. 현재 준비 상태

- Google Play store listing draft: 준비 완료
- Google Play data safety draft: 준비 완료
- 개인정보 처리방침 URL readiness: 준비 완료
- Android Debug Build artifact: 생성 가능
- 실제 Android 기기 QA: Blocked
- 실제 스크린샷 이미지: 아직 생성하지 않음
- release build/signing/AAB: 미진행
- 실제 광고 SDK/결제 SDK: 미연동

## 3. 스크린샷 후보 화면

| 우선순위 | 화면 | 목적 | 표시하면 좋은 내용 | 주의사항 |
| --- | --- | --- | --- | --- |
| 1 | 홈 화면 | 앱 첫인상과 오늘의 흐름 요약 | 오늘의 운세, 사주 요약, 루틴 카드 | 개인정보가 보이지 않게 샘플 데이터 사용 |
| 2 | 오늘의 운세 상세 | 핵심 콘텐츠 확인 | 총운/재물/연애/직장/건강/학업 카드 | 단정적 예언 문구 피하기 |
| 3 | 사주 인사이트 | 앱 차별점 강조 | 오행 요약, 강점/주의점 | 공포/불안 조장 문구 피하기 |
| 4 | 저장한 풀이 | 재방문 가치 강조 | 저장 목록, 다시 보기 | 생년월일/시간 등 민감 정보 노출 금지 |
| 5 | 개인정보 안내 | 신뢰성 강조 | localStorage, 동의 설정 안내 | 너무 복잡하지 않게 |
| 6 | 동의 설정 | 광고/분석/개인화 준비 상태 안내 | 선택 동의 UI | 실제 SDK 미연동 상태와 충돌 없게 |
| 7 | 방문 streak/루틴 | 매일 사용 동기 강조 | streak, 오늘의 루틴 | 과도한 중독 유도 문구 피하기 |

## 4. 권장 스크린샷 세트

최소 권장 세트:

1. 홈 화면
2. 오늘의 운세 상세
3. 사주 인사이트
4. 저장한 풀이
5. 개인정보 안내 또는 동의 설정

확장 세트:

6. 방문 streak/루틴
7. 프로필 입력/온보딩
8. 보상형 상세 콘텐츠 안내 화면

## 5. 스크린샷 촬영 기준

- 실제 사용자의 생년월일, 출생시간, 성별이 보이지 않게 합니다.
- 테스트용 샘플 프로필을 사용합니다.
- 스크린샷 내 문구는 참고용 콘텐츠임을 해치지 않아야 합니다.
- 과도한 성공 보장, 재물 보장, 연애 성사 보장 문구를 사용하지 않습니다.
- 의료, 법률, 투자, 진로 결정을 대신하는 표현을 사용하지 않습니다.
- 개인정보 안내 화면은 localStorage 중심 저장 구조와 실제 SDK 미연동 상태를 반영해야 합니다.
