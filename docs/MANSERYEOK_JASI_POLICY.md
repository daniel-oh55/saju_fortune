# MANSERYEOK_JASI_POLICY

## 사용자 계산 기준 표시

- 23시 이후 사용자는 선택한 기준이 실제 계산에 어떻게 반영되었는지 확인할 수 있어야 한다.
- SettingsPage에 사주 계산 기준 요약을 표시한다.
- `same_day` / `next_day` 선택값과 `convertedSolar`를 함께 보여주어 혼동을 줄인다.
- 이 표시는 정책 확정이 아니라 사용자 투명성 개선이다.

## lateNightJasiPolicy 회귀 검증

- `same_day`는 23:00~23:59 입력을 입력한 날짜 기준으로 계산한다.
- `next_day`는 23:00~23:59 입력을 다음 날 00:분 기준으로 계산한다.
- 22시대 이하 입력에는 `next_day`가 적용되지 않는다.
- `birthTimeUnknown=true`인 경우 `next_day`가 적용되지 않는다.
- `same_day`와 `next_day`는 서로 다른 profileId를 생성해야 한다.
- 이 검증은 외부 기준값과의 pass/fail 판정이 아니라 사용자 선택 정책이 의도대로 동작하는지 확인하는 회귀 테스트다.

## 2026-06-09 사용자 선택 정책

- 23:00~23:59 출생자는 프로필 입력 화면에서 자시 기준을 선택할 수 있다.
- 기본값은 `same_day`이며, 기존 계산 결과와 기존 사용자 흐름을 유지하기 위한 선택이다.
- `same_day`는 입력한 생년월일을 그대로 기준으로 풀이한다.
- `next_day`는 사용자가 명시적으로 선택한 경우에만 23시 이후 출생 시간을 다음 날 00시대로 보정해 계산한다.
- 이 선택지는 23시 이후 기준 차이를 투명하게 안내하기 위한 임시 정책이며, 최종 자시 정책 확정은 아니다.
- 세 번째 외부 만세력 기준과 추가 샘플 검증이 쌓이면 기본값과 엔진 정책을 다시 검토한다.

## 사용자 안내 정책

- 23:00~23:59 출생자는 자시 기준에 따라 일주/시주가 달라질 수 있다.
- production 엔진 정책 확정 전까지는 사용자에게 안내 문구를 표시한다.
- 안내 문구는 불안감을 주지 않고, 기준 차이가 있을 수 있음을 설명하는 수준으로 유지한다.
- 향후 정책 확정 후 같은 날짜 기준 / 다음 날 자시 기준 / 사용자 선택 기준 중 하나를 반영할 수 있다.

## 외부 기준 비교 결과

### 1990-02-03 23:30

현재 후보 결과:

- original/sameDayJasi: 기사 / 정축 / 기해 / 병자
- nextDayCandidate/midnightReference: 기사 / 정축 / 경자 / 병자

외부 기준:

- sky.told.me: 기사 / 정축 / 경자 / 병자
- 포스텔러: 기사 / 정축 / 기해 / 을해

관찰:

- sky.told.me는 nextDayCandidate/midnightReference와 일치한다.
- 포스텔러는 현재 후보 중 완전 일치하는 값이 없다.
- original/sameDayJasi는 포스텔러와 일주는 같지만 시주가 다르다.
- 따라서 23시 정책을 확정하기 전에 세 번째 기준 만세력 확인이 필요하다.

현재 결론:

- 아직 production 엔진에 자시 정책을 반영하지 않는다.
- `solar_before_ipchun`은 `reference_conflict`와 expected null을 유지한다.
- 세 번째 기준 확인 후 다음 날 자시 기준 또는 사용자 선택 옵션을 검토한다.

## 목적

23시 이후 출생 시간의 자시/야자시/조자시 기준을 정리하기 위한 조사 문서다.
현재 단계에서는 production 엔진을 변경하지 않고, 기준 후보와 외부 기준값 차이를 비교한다.

## 현재 문제

23:00~23:59 출생은 만세력 서비스나 명리 기준에 따라 일주/시주가 달라질 수 있다.

예시 샘플:

- sampleId: `solar_before_ipchun`
- 입력: `1990-02-03 23:30`
- sky.told.me 기준: 기사년 정축월 경자일 병자시
- 포스텔러 기준: 기사년 정축월 기해일 을해시
- 현재 상태: `reference_conflict`
- expected: null

## 정책 후보

### 후보 A. 같은 날짜 기준

23:00~23:59를 입력 날짜의 밤으로 보고, 일주는 같은 날짜 기준으로 유지한다.

장점:

- 사용자가 입력한 생년월일과 직관적으로 맞다.
- 날짜 변환 설명이 적다.

단점:

- 일부 만세력 서비스의 자시 처리와 다를 수 있다.

### 후보 B. 다음 날 자시 기준

23:00~23:59를 다음 날 자시로 보고, 일주가 다음 날 기준으로 넘어갈 수 있다.

장점:

- 일부 전통 명리 기준과 맞을 수 있다.
- 자시를 하루의 시작으로 보는 방식과 일치할 수 있다.

단점:

- 사용자가 입력한 날짜와 계산 일주가 달라져 설명이 필요하다.

### 후보 C. 사용자 선택 기준

23시 이후 출생인 경우 "같은 날짜 기준"과 "다음 날 자시 기준" 중 사용자가 선택하도록 한다.

장점:

- 기준 차이를 투명하게 처리할 수 있다.
- 외부 만세력과 비교할 때 유연하다.

단점:

- MVP 입력 화면이 복잡해질 수 있다.

## 현재 원칙

- v0 엔진에서는 23시 이후 자시 정책을 확정하지 않는다.
- `reference_conflict` 샘플은 expected를 입력하지 않는다.
- 외부 기준값이 충돌하는 경우 pass/fail로 판단하지 않는다.
- 최소 1개 이상의 추가 한국 만세력 기준을 확인한 뒤 정책을 결정한다.
